# Solução para Erro 403 no cPanel

## 🚨 Problema
Ao acessar o site, ocorre redirecionamento para `403.shtml/`

## 🔧 Soluções

### **Solução 1: Verificar Permissões dos Arquivos**

1. **Acesse o File Manager no cPanel**
2. **Navegue até `public_html`**
3. **Selecione todos os arquivos e pastas**
4. **Clique em "Change Permissions"**
5. **Defina as permissões:**
   - **Arquivos HTML**: 644
   - **Arquivos CSS/JS**: 644
   - **Pasta `_next`**: 755
   - **Pasta `simulador-iuc`**: 755
   - **Pasta `simulador-isv`**: 755

### **Solução 2: Verificar o Arquivo .htaccess**

1. **No File Manager, procure o arquivo `.htaccess`**
2. **Se não existir, crie um novo arquivo chamado `.htaccess`**
3. **Cole o seguinte conteúdo:**

```apache
# Configuração para Next.js Static Export
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule ^(.*)$ index.html [QSA,L]

# Configuração de cabeçalhos de segurança
<IfModule mod_headers.c>
    Header always set X-Content-Type-Options "nosniff"
    Header always set X-Frame-Options "SAMEORIGIN"
    Header always set X-XSS-Protection "1; mode=block"
</IfModule>

# Configuração de cache
<IfModule mod_expires.c>
    ExpiresActive On
    ExpiresByType text/css "access plus 1 month"
    ExpiresByType text/javascript "access plus 1 month"
    ExpiresByType application/javascript "access plus 1 month"
    ExpiresByType image/jpg "access plus 1 year"
    ExpiresByType image/jpeg "access plus 1 year"
    ExpiresByType image/gif "access plus 1 year"
    ExpiresByType image/png "access plus 1 year"
</IfModule>
```

### **Solução 3: Verificar o Arquivo index.html**

1. **Confira se o arquivo `index.html` existe em `public_html`**
2. **Verifique se o conteúdo está correto (não está vazio)**
3. **Se necessário, reextraia o ZIP**

### **Solução 4: Verificar Configurações do cPanel**

1. **Acesse "MultiPHP Manager" no cPanel**
2. **Selecione seu domínio**
3. **Configure:**
   - **PHP Version**: 7.4 ou superior
   - **Engine**: On

### **Solução 5: Verificar Configurações de Segurança**

1. **Acesse "ModSecurity" no cPanel**
2. **Verifique se há regras bloqueando o acesso**
3. **Se necessário, desative temporariamente para testar**

## 🛠️ Comandos Úteis (via SSH)

Se tiver acesso SSH:

```bash
# Verificar permissões
ls -la /home/usuario/public_html/

# Corrigir permissões
chmod 644 /home/usuario/public_html/*.html
chmod 644 /home/usuario/public_html/*.css
chmod 644 /home/usuario/public_html/*.js
chmod 755 /home/usuario/public_html/_next
chmod 755 /home/usuario/public_html/simulador-iuc
chmod 755 /home/usuario/public_html/simulador-isv

# Verificar se o arquivo .htaccess existe
ls -la /home/usuario/public_html/.htaccess
```

## 📋 Checklist de Verificação

- [ ] Permissões dos arquivos corretas
- [ ] Arquivo `.htaccess` presente e configurado
- [ ] Arquivo `index.html` existe e não está vazio
- [ ] PHP habilitado no cPanel
- [ ] ModSecurity não bloqueando
- [ ] Cache do navegador limpo (Ctrl+F5)

## 🆘 Suporte Técnico

Se o problema persistir:

1. **Verifique os logs de erro no cPanel**
   - Vá até "Errors" no cPanel
   - Verifique mensagens de erro recentes

2. **Teste com um arquivo HTML simples**
   - Crie um arquivo `test.html` com conteúdo básico
   - Tente acessar `https://seudominio.com/test.html`

3. **Contate o suporte da hospedagem**
   - Informe o erro 403
   - Pergunte sobre configurações de segurança

## 🎯 Solução Rápida

**Para resolver imediatamente:**

1. **Reextraia o ZIP no cPanel**
2. **Defina permissões:**
   - Arquivos: 644
   - Pastas: 755
3. **Crie o arquivo `.htaccess`** (se não existir)
4. **Limpe o cache do navegador** (Ctrl+F5)
5. **Teste novamente**

**Tempo estimado:** 5 minutos
